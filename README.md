# Architectural Review Template

This repository contains a collection of templates and checklists to support consistent, high-quality architecture reviews across software projects.

## Why Use This?

Architectural decisions shape the long-term health and scalability of a system. These templates help:
- Standardize the review process
- Ensure alignment with business and technical goals
- Document key decisions and trade-offs

## Contents

- `templates/` — Reusable templates for design documentation and review
- `examples/` — Sample filled-out documents
- `checklists/` — Review guides and quality gates

## How to Use

1. Copy the relevant template from `templates/` into your project or wiki.
2. Customize it based on your domain and system.
3. Use the checklists during design reviews to ensure completeness.
4. Store completed docs in your architecture repo or knowledge base.

## Templates Included

- High-Level Design Template
- Architecture Decision Record (ADR)
- Definition of Done (Architecture-focused)
- Architecture Review Checklist
